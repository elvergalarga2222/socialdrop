# PR #03 — Mejores horas para publicar (basado en historial)

**Área:** Dashboard
**Prioridad:** 🟢 Mejora
**Esfuerzo estimado:** M (2-8h)

**Problema:**
`BrandProfile.optimalTimes` existe pero se llena manualmente. No se calcula desde el rendimiento real del usuario.

**Solución propuesta:**
Endpoint `GET /api/stats/best-times?platform=` que agrupe `PostAnalytics` (join a `PostIntegration.publishedAt`) por `dayOfWeek` × `hourBucket` y promedie `engagementRate`. Devolver top-3 franjas. Mostrar heatmap 7×24 en el dashboard e integrarlo como sugerencia al crear slots en `/queue`. Umbral mínimo de N posts por celda para no recomendar con ruido.

**Archivos a modificar:**
- `apps/api/src/modules/stats/stats.service.ts` (+ controller)
- `apps/web/src/app/page.tsx`
- (opcional) `apps/web/src/app/queue/page.tsx` (sugerencias de slots)

**Criterios de aceptación:**
- [ ] Recomienda solo franjas con ≥5 publicaciones
- [ ] Heatmap accesible con valores numéricos visibles
