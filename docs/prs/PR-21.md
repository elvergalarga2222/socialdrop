# PR #21 — Notificación de "reconectar cuenta" al fallar refresh de token

**Área:** Seguridad / UX
**Prioridad:** 🟡 Importante
**Esfuerzo estimado:** M (2-8h)

**Problema:**
Cuando el refresh de token falla de forma no recuperable (IG long-lived caducado por inactividad, YouTube revocado — el bug conocido), el post queda en ERROR pero **el usuario no se entera de que debe reconectar la cuenta**. El post seguirá fallando hasta que alguien mire los debug logs.

**Solución propuesta:**
1. Campo `Integration.needsReauth Boolean @default(false)`.
2. Marcarlo al capturar `RefreshTokenError` no recuperable en el processor o en metrics.
3. Banner persistente en dashboard + estado "Reconectar" en `/integrations`; limpiar el flag al completar OAuth de nuevo.
4. Integrar con alertas (PR #23) para notificar por canal externo.

**Archivos a modificar:**
- `libs/prisma/prisma/schema.prisma` (+ migración)
- `apps/api/src/modules/posts/processors/post-scheduler.processor.ts`
- `apps/api/src/modules/metrics/metrics.service.ts`
- `apps/api/src/modules/integrations/integrations.controller.ts`
- `apps/web/src/app/integrations/page.tsx`, `apps/web/src/app/page.tsx`

**Criterios de aceptación:**
- [ ] Refresh fallido no recuperable marca la integración
- [ ] UI muestra "Reconectar" y el flujo OAuth lo limpia
