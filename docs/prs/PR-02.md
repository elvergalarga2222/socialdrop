# PR #02 — Gráfica de rendimiento por plataforma (+ fix N+1 en getByPlatform)

**Área:** Dashboard
**Prioridad:** 🟢 Mejora
**Esfuerzo estimado:** M (2-8h)

**Problema:**
No hay visualización comparativa entre Instagram/Facebook/YouTube. `StatsService.getByPlatform` existe pero carga **todos los posts de cada integración en memoria y los filtra en JS** (N+1 latente) y no se usa en el home.

**Solución propuesta:**
Reescribir `getByPlatform` con `groupBy(['platform','status'])` de Prisma (una sola query agregada). Añadir al dashboard barras apiladas (publicados/pendientes/fallidos) por plataforma + engagement medio por plataforma (desde `PostAnalytics`). Usar SVG inline o Recharts, con etiquetas directas (la identidad no puede depender solo del color).

**Archivos a modificar:**
- `apps/api/src/modules/stats/stats.service.ts`
- `apps/web/src/app/page.tsx` (o nuevo `apps/web/src/components/PlatformBreakdownChart.tsx`)

**Criterios de aceptación:**
- [ ] `getByPlatform` usa `groupBy` (sin cargar posts en memoria)
- [ ] Gráfica accesible (etiquetas directas, no solo color)
- [ ] Respeta dark/light theme (ThemeProvider existente)
