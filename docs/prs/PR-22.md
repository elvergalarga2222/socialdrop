# PR #22 — 🔴 Health checks para API/DB/Redis/colas

**Área:** Infraestructura / Observabilidad
**Prioridad:** 🔴 Crítico
**Esfuerzo estimado:** S (< 2h)

**Problema:**
No hay endpoint de salud monitoreable desde fuera del VPS (`main.ts` solo expone `GET /` hello). Una caída de Postgres/Redis/worker pasa inadvertida hasta que un usuario reporta.

**Solución propuesta:**
`@nestjs/terminus`: `GET /api/health` que chequee Postgres (Prisma `SELECT 1`), Redis (ping) y la conexión de BullMQ (worker activo / profundidad de cola). Añadir `healthcheck` del contenedor `api` en docker-compose. Configurar un uptime monitor externo (UptimeRobot o similar) apuntando al endpoint.

**Archivos a modificar:**
- Nuevo `apps/api/src/app/health.controller.ts` (+ módulo terminus)
- `docker-compose.yml`

**Criterios de aceptación:**
- [ ] `/api/health` devuelve 200 con estado de DB/Redis/cola; 503 si algo cae
- [ ] Healthcheck de contenedor configurado
