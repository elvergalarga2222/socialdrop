# PR #36 — Aplicación del sistema: AppShell, dashboard, calendario, cola y tablas

**Área:** Diseño / Frontend
**Prioridad:** 🟡 Importante
**Esfuerzo estimado:** L (1-2 días)
**Depende de:** PR #33 (tokens), PR #34 (motion), PR #35 (PlatformChip)

**Problema:**
Los componentes actuales usan `bg-gray-900 border border-gray-800 rounded-xl` repetido a mano por pantalla (con variantes light/dark dobles), fuente del sistema para todo (números incluidos), bordes visibles como único mecanismo de separación, y estilos de tabla inconsistentes. Nada consume tokens porque no existían.

**Solución propuesta:**
Migración sistemática al sistema de PR #33 (buscar-y-reemplazar guiado, pantalla por pantalla):

1. **AppShell:** sidebar y header sobre `--color-base`; item activo = pill (`--radius-pill`) con `--color-accent` (se conserva el violeta actual); desaparecen `border-r`/`border-b` — la separación es el salto de superficie. Labels de sección (AUTOMATIZACIÓN, CONFIGURACIÓN) en label-11 uppercase muted.
2. **Cards (dashboard y todas las vistas):** `bg-surface rounded-card` **sin borde ni sombra**; hover interactivo = `surface-2`. Stat cards del dashboard: label en Satoshi 12.5 muted, cifra en JetBrains Mono 32 `tabular-nums`, delta con ▲lima/▼ámbar (signo + color, nunca solo color). Entrada con `Stagger`+`FadeUp` (PR #34).
3. **Tratamiento de tablas** (dashboard "Posts Recientes", competitors, listas): estilo instrumento — sin bordes verticales ni zebra; divisores horizontales `rgba(255,255,255,0.04)`; header 11px uppercase tracking-wide muted; números/fechas en mono 12 alineados a la derecha; row hover `surface-2`; wrapper `overflow-x-auto` (coordinado con PR #05, que sigue vigente).
4. **StatusBadge:** pill (`--radius-pill`) con ícono + label: ✓ PUBLICADO lima, ○ PROGRAMADO violeta, ⚠ ERROR ámbar, ─ BORRADOR muted, ▸ PUBLICANDO violeta pulsante (única animación de estado permitida).
5. **Calendario:** celdas sin bordes duros (grid lines `rgba(255,255,255,0.04)`), día actual = número en pill accent; pills de post → `PlatformChip sm` + hora en mono 11; drag & drop conserva su comportamiento.
6. **Cola:** tabs de plataforma → `PlatformChip md` + label; celdas del grid horario: inactiva `surface`, hover `surface-2`, activa `accent` al 24% con punto accent; hora/día en mono.
7. **Botones:** primario = pill `--color-accent`; secundario = pill `surface-2`; destructivo = pill ámbar-outline (el ámbar es el negativo del sistema; no se introduce un rojo extra — presupuesto de 3 acentos).

**Archivos a modificar:**
- `apps/web/src/components/AppShell.tsx`, `StatusBadge.tsx`, `EditPostModal.tsx`, `QuickUploadModal.tsx`
- `apps/web/src/app/page.tsx`, `calendar/page.tsx`, `queue/page.tsx`, `competitors/page.tsx` (tabla), `inbox/page.tsx`
- Barrido de `gray-800|gray-900|border-gray|indigo-[0-9]` → tokens (grep de verificación en CI del PR)

**Criterios de aceptación:**
- [ ] Cero `bg-gray-*`/`border-gray-*`/hex crudo en los archivos tocados (solo utilidades de token)
- [ ] Cards sin borde; jerarquía legible solo por superficies (base→surface→surface-2)
- [ ] Toda cifra de datos en mono `tabular-nums`; labels en Satoshi; headers en Space Grotesk
- [ ] Tablas con el tratamiento instrumento (sin bordes verticales, header uppercase, hover)
- [ ] Estados siempre ícono+label+color; foco visible con anillo accent en elementos interactivos
- [ ] Sin regresión responsive (PR #05 sigue aplicando)

**Notas técnicas:**
Migrar por pantalla en commits separados (AppShell → dashboard → calendario → cola → tablas) para poder revisar visualmente cada paso. Las clases `dark:` duplicadas se eliminan (dark-first de PR #33). No tocar lógica ni data-fetching: este PR es exclusivamente presentacional.
