# PR #17 — Flow/DM/inbox usan token por-integración (no env global)

**Área:** Feature / Seguridad
**Prioridad:** 🟡 Importante
**Esfuerzo estimado:** M (2-8h)

**Problema:**
`FlowEngine.sendDM/replyComment` e `InboxService.sendReply` usan `FACEBOOK_ACCESS_TOKEN` de **env global** → solo funcionan para una cuenta y son incompatibles con multi-usuario. Además `WebhookService.processEvent` busca flows sin resolver a qué usuario pertenece el evento (`flow.findMany` sin userId), y los `fetch` de envío no comprueban `res.ok` (fallos silenciosos).

**Solución propuesta:**
1. Resolver la `Integration` a partir del `entry.id` (página/cuenta) del webhook y usar **su** token (descifrado vía PR #15).
2. Enrutar el evento solo a los flows del `userId` dueño de esa integración.
3. Comprobar `res.ok` en `sendDM`/`replyComment`/`sendReply` y registrar el error.

**Archivos a modificar:**
- `apps/api/src/modules/flows/flow.engine.ts`
- `apps/api/src/modules/inbox/inbox.service.ts`
- `apps/api/src/modules/webhooks/webhook.service.ts`

**Criterios de aceptación:**
- [ ] DMs/replies usan el token de la integración del usuario dueño del evento
- [ ] Un evento de la página X no dispara flows de otros usuarios
- [ ] Errores de envío quedan registrados (no silenciosos)
